# Dusk v3 Design

## Motivation

Dusk started as an academic translation of Meeus's _Astronomical Algorithms_ into Go. It was forked and reshaped into a usable library (v2). v3 cleans the API surface, consolidates files, fixes correctness gaps, and removes anything that doesn't earn its place.

Goals: clean, simple, as small as is reasonable. Clean interfaces, usable, correct.

Non-goals: new features, performance optimization, broader astronomical coverage.

No known external consumers. Breaking changes are unconstrained.

## Public API

### Constructor

```go
func NewObserver(lat, lon float64, loc *time.Location) (Observer, error)
```

Validates lat `[-90, 90]`, lon `[-180, 180]`, non-nil loc, rejects NaN/Inf. Returns the only way to obtain a valid `Observer`. Fields are unexported.

### Functions

```go
func SunriseSunset(date time.Time, obs Observer) (SunEvent, error)
func CivilTwilight(date time.Time, obs Observer) (TwilightEvent, error)
func NauticalTwilight(date time.Time, obs Observer) (TwilightEvent, error)
func AstronomicalTwilight(date time.Time, obs Observer) (TwilightEvent, error)
func MoonriseMoonset(date time.Time, obs Observer) (MoonEvent, error)
func LunarPhase(t time.Time) LunarPhaseInfo
```

### Errors

```go
var ErrCircumpolar = errors.New("dusk: object is circumpolar (always above the horizon)")
var ErrNeverRises  = errors.New("dusk: object never rises at this latitude")
```

### Removed from public API

Everything else. Specifically:

- `ObjectTransit`, `Transit` type
- `SolarPosition`, `LunarPosition`, `LunarEclipticPosition`
- `EclipticToEquatorial`, `EquatorialToHorizontal`
- `HourAngle`, `AngularSeparation`
- `JulianDate`, `ValidJulianDateRange`, `LocalSiderealTime`
- `ErrDateOutOfRange`
- `Equatorial`, `Horizontal`, `Ecliptic` types (unexported, still used internally)
- `Observer.Elev` field (removed entirely, elevation correction deleted)

## Validation and Error Handling

### Observer validation at construction

`NewObserver` is the single validation boundary. It rejects:

- `nil` location
- Lat outside `[-90, 90]`
- Lon outside `[-180, 180]`
- NaN or Inf in lat or lon

Public functions do not re-validate the observer. If you have an `Observer`, it's valid.

### Date range validation at entry points

Every public function that takes `date time.Time` checks `validJulianDateRange(date)` before computing. The check and its error are unexported â€” callers see a clear message but can't match with `errors.Is`.

### "No event" contract

Two patterns, serving different purposes:

- **Sentinel errors** (`ErrCircumpolar`, `ErrNeverRises`) â€” the geometry makes the event impossible. The caller needs to know _why_.
- **Zero-value `time.Time`** â€” one event of a pair didn't occur during the scan window (e.g., Moon rose but didn't set before midnight). No error; check with `.IsZero()`.

The package doc explains this distinction explicitly.
